The trust path

The key architecture change was not a new service. It was one deterministic gate in the existing file pipeline.

External feedsFIFA data with a fallback source
›
Fetch scriptNormalizes matches and writes JSON
›
ValidatorRejects malformed or inconsistent data
›
GitHub PagesPublishes the last valid static files
ELI5: The fetch script is a delivery truck. The validator is the person at the loading dock who checks every box before it reaches the store shelf.

Why time logic stayed outside the render snapshot

The freshness warning lives in `main.js`. A label such as "updated 42 minutes ago" changes as the clock moves. Putting it inside byte-locked render output would make a stable fixture fail for the passage of time rather than a code regression.

General rule: frozen snapshots need frozen inputs. Keep live-clock presentation outside deterministic output, or inject a test clock.

Why flags were low risk

The builder already had `flagImg()` and all SVG files. The map reused the same helper. Unknown names still return an empty string. Placeholder labels do not pretend to be countries. The change added no service, package, or mapping copy.

Learning notes

Each note ties one repository decision to a reusable engineering principle.

1Validate the artifact, not the process

A fetch command exiting with code 0 proves it ran. It does not prove the resulting JSON makes sense. Validate the file downstream code will consume.

ELI5: A cook saying "done" does not prove the meal is safe. Look at the plate.
2More automation needs a stronger gate

Half-hour runs increase speed and failure opportunities. The validator turns higher frequency into useful leverage instead of higher publishing risk.

3A stale truth still looks like a lie

Correct old data with no timestamp warning damages trust. Keep showing the last known result, then state its age plainly.

4Reuse beats parallel systems

The flag helper already knew filenames, aliases, accessibility text, and missing-name behavior. Reusing it prevented two country maps from drifting.

5Failure UI is product UI

A blank page and a silent save failure are not technical details. They are user-facing states. Name the problem and give the next action.

6Edit voice after behavior settles

Copy changes across golden output are easier to review once the data and interaction work is stable. One final pass keeps wording consistent and snapshot changes explainable.
