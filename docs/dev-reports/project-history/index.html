The current build process

Formal multi-plan work now carries intent and evidence from the first sentence through the public report.

1. GCSEGoal, Context, Source, Expectations
2. BriefContract, scope, risks, acceptance
3. PlansRanked files, steps, edge cases, tests
4. BuildSmall checkpoints and bounded commits
5. ProveTests, validators, browser evidence
6. TeachResults and rendered report

The plan is part of the product. A future contributor should see why a decision exists, what was deliberately omitted, and which evidence allowed it to ship.

Protect intent

Preserve original briefs and plans. Mark reconstructions. Never invent missing history.

Protect behavior

Use pure helpers, frozen inputs, golden output, browser checks, and generated-data gates.

Protect ownership

Preview locally, keep personal data local, and require explicit approval for outward changes.
